# 第二章 概念框架

引言指出，当前研究者在使用大型语言模型（LLM）进行学术写作时缺乏系统化的能力认知与工具选择依据。为回应这一痛点，本章首先建立学术写作的任务分类体系，继而定义用于评估 LLM 表现的多维指标，最终将二者融合为"任务类型 $\times$ 能力维度"的二维评估矩阵，作为全文实证检验的分析基础。

## 2.1 学术写作任务的类型学划分

学术写作并非单一、同质的语言生成活动，而是由若干认知要求和产出目标各异的子任务构成的复合过程。已有研究指出，写作的本质是"问题解决"（Flower & Salab, 1996），而问题的形态直接取决于研究者所处的工作阶段。本文沿袭研究流程的自然分段思路，将学术写作划分为四个可操作的子任务：文献综述撰写、方法描述、结果解释与讨论撰写。这一划分既覆盖了从前期准备到成果产出的完整链条，又确保每个子任务在认知负荷和语言风格上具有可辨识的差异。

**（1）文献综述撰写。** 该子任务的核心是对已有研究进行系统性检索、筛选、综合与批判性评价。其认知要求集中在信息提取、主题归纳与论证脉络的构建上，而非原创性论点的提出 [R9]。在语言风格层面，文献综述以述评结合为特征，需要在客观陈述（"某某学者发现……"）和主观评判（"然而，该研究的样本量存在局限"）之间频繁切换。这种双重语体的交替使用对 LLM 而言构成显著挑战：模型倾向于过度依赖表面词汇匹配而非语义层面的批判性整合 [R8]。

**（2）方法描述。** 该方法论子任务的目标是精确、可复现地呈现研究设计、数据收集流程与分析技术。其认知核心在于技术细节的完整性和逻辑顺序的严格性，语言风格要求客观、精确、避免歧义，通常大量使用被动语态和公式化表达。与文献综述不同，方法描述不需要批判性综合，但要求对专业术语和技术参数的零误差呈现。已有研究表明，LLM 在结构化技术文本生成中表现相对稳定 [R4]，但其对实验设计合理性的内在判断能力仍待验证。

**（3）结果解释。** 该子任务要求将数据分析所得的发现转化为有意义的学术论断，即在"数据说明了什么"与"这些发现意味着什么"之间建立逻辑桥梁。其认知核心是推理能力——从具体数值或模式中提取一般性结论，同时避免过度泛化。语言风格上需要在客观呈现（"图 3 显示……"）和适度推断（"这一趋势可能反映……"）之间保持平衡。结果解释是学术写作中最易出现逻辑跳跃的环节 [R5]，也是 LLM 幻觉（hallucination）风险最高的子任务之一。

**（4）讨论撰写。** 作为论文的最后正文部分，讨论需要将研究结果置于更广阔的学术语境中，回应引言提出的研究问题，阐明理论贡献与实践启示，并承认研究的局限性。其认知要求最高——需要在开放性和约束性之间取得平衡：既不能局限于重复发现，也不能脱离数据空泛议论 [R2]。语言风格上要求多层次论证能力的综合运用，涉及让步、对比、归纳和演绎等多种修辞策略的交织。

上述四类子任务并非孤立存在，而是在研究流程中形成递进关系：文献综述为方法选择提供依据，方法描述为结果呈现奠定技术基础，结果解释将数据转化为发现，讨论撰写则将这些发现在更广泛的学术对话中进行定位。这种递进性意味着不同子任务对 LLM 的认知要求呈递增趋势——从信息整合（文献综述）到技术精确性（方法描述），再到推理能力（结果解释），最终抵达综合论证（讨论）。为系统评估 LLM 在上述各子任务中的表现差异，需要建立一套多维度的能力评价指标体系。

**本节摘要**：本章将学术写作划分为文献综述撰写、方法描述、结果解释与讨论撰写四个子任务，指出每一类在认知要求和语言风格上存在本质差异，且四类任务沿研究流程呈递进关系。这一分类为后续引入 LLM 评估维度提供了"任务轴"的基础。下一节将从"写什么"转向"如何衡量 AI 在这些任务上的表现"，定义四个能力评估维度。

## 2.2 LLM 能力维度的操作化定义

在确立学术写作的任务分类之后，需要建立一套可量化、可复现的能力评估指标体系，用于衡量 LLM 在各子任务中的实际表现水平。单一维度的评价（如语言流畅度或文本连贯性）已被证明会严重高估模型的真实学术能力 [R6]。本文提出四个相互独立又互补的评估维度：准确性、逻辑连贯性、学术规范合规度和创造性，以此构建多维综合的评价框架。

**（1）准确性（Accuracy）。** 该维度衡量 LLM 生成的内容在事实层面和概念层面的正确程度。具体包括三个方面：其一，事实准确性——所陈述的研究发现、数据数值、理论名称等是否与原始来源一致 [R8]；其二，概念准确性——专业术语的使用是否恰如其分，理论框架的描述是否忠实于原初定义；其三，技术准确性——公式推导、统计方法描述是否符合学科规范。在操作化层面，准确性可通过与人工专家标注的"金标准"文本进行逐项比对来计算准确率指标 $A = \frac{n_c}{n_t}$，其中 $n_c$ 为正确陈述数，$n_t$ 为总陈述数。已有研究指出，LLM 在事实性内容生成中容易出现"自信的幻觉"——以高度确定性的语气呈现错误信息 [R10]，这一现象在学术写作者身上可能产生更为严重的后果。

**（2）逻辑连贯性（Logical Coherence）。** 该维度评估文本内部论证链条的完整性与推理过程的严密性。其操作化包含两个层次：段落内连贯性和跨段落连贯性。前者关注句子之间的衔接手段是否恰当，如连接词的使用、代词指代的清晰性以及话题延续的一致性；后者考察论点在不同段落间的推进是否具有累积效应而非简单重复 [R5]。逻辑连贯性的测量可采用基于论证图（argument map）的结构化评估方法：首先识别文本中的所有论断及其支撑关系，然后检验是否存在逻辑跳跃、循环论证或证据链断裂。对于 LLM 而言，长上下文窗口虽然提升了全局一致性能力，但在需要多步推理的复杂论证场景中仍可能出现中间步骤缺失 [R4]。

**（3）学术规范合规度（Academic Convention Compliance）。** 该维度衡量文本是否符合目标学科共同体认可的写作规范与格式要求。具体涵盖五个方面：引文格式的准确性、参考文献列表的完整性、图表标注的规范性、段落结构的合理性以及语域的适切性 [R7]。在引文处理方面，已有专门研究探讨 LLM 在技术写作中的参考信息处理能力，发现模型在生成准确且格式规范的引用时存在系统性偏差 [R8]。学术规范合规度的评估可通过制定清单式评分量表实现，每一项规范要求的满足程度以二元（符合/不符合）或三级（完全符合/部分符合/不符合）方式编码。需要指出的是，不同学科领域的规范要求存在显著差异——例如工程类论文偏好简洁直接的技术报告体例，而人文类论文则允许更为灵活的论证结构 [R9]，因此在实际评测中需以目标学科的规范为参照基准。

**（4）创造性（Creativity）。** 该维度评估 LLM 在学术写作中生成新颖且有价值的观点、分析方法或理论洞见的能力。需要强调的是，此处的"创造性"并非指文学意义上的想象力和修辞技巧 [R6]，而是指在严格约束的学术语境中提出有意义的新视角、建立跨领域关联或识别现有研究盲区的认知能力 [R2]。操作化层面可采用三项指标：其一，观点新颖性——生成的论点是否超越了对已有文献的简单综合；其二，方法创新性——对研究设计或分析策略的建议是否具有启发性；其三，理论贡献意识——能否主动将研究发现与更广泛的理论框架相连接 [R1]。创造性维度的评估最具主观性，建议采用专家评分法并结合共识协议（如 Delphi 法）以降低评价者间差异。

上述四个维度并非同等重要——在不同子任务中其权重分配应有所调整。例如，方法描述对准确性和规范合规度要求极高，而对创造性的容忍度较低；讨论撰写则相反，在保持准确性和逻辑连贯性的基础上高度依赖创造性 [R3]。这种动态权重关系进一步印证了二维评估矩阵的必要性：单一维度的能力画像无法捕捉 LLM 在不同写作情境中的真实表现模式。

**本节摘要**：本章定义了准确性、逻辑连贯性、学术规范合规度和创造性四个相互独立的 LLM 评估维度，给出每个维度的操作化定义和测量方法，并指出不同子任务中各维度的重要性权重存在差异。结合前节的任务分类与本节的能力维度，下一节将二者融合为"任务类型 $\times$ 能力维度"的二维评估矩阵框架。

## 2.3 二维评估矩阵的构建

基于前文建立的任务分类体系（四个子任务）和能力维度定义（四项指标），可将二者交叉组合为一个 $4 \times 4$ 的二维评估矩阵。该矩阵的行代表学术写作的四个子任务，列代表 LLM 能力的四个评估维度，共计十六个单元格，每个单元格描述在特定子任务和特定维度的交集处对 LLM 表现的预期水平与评估重点。这一结构不仅为实证检验提供了系统化的分析框架，也可作为未来模型迭代过程中追踪能力演进趋势的基准工具。

矩阵的结构如下（表 2-1）：

**表 2-1 "任务类型 $\times$ 能力维度"二维评估矩阵**

| 子任务 \ 维度 | 准确性 (A) | 逻辑连贯性 (C) | 学术规范合规度 (F) | 创造性 (R) |
|---|:--:|:--:|:--:|:--:|
| 2.1 文献综述撰写 | $H$（高预期） | $M$（中预期） | $H$（高预期） | $L$（低预期） |
| 2.2 方法描述 | $VH$（极高预期） | $M$（中预期） | $VH$（极高预期） | $VL$（极低预期） |
| 2.3 结果解释 | $H$（高预期） | $H$（高预期） | $M$（中预期） | $M$（中预期） |
| 2.4 讨论撰写 | $H$（高预期） | $VH$（极高预期） | $M$（中预期） | $H$（高预期） |

注：$VL$ = 极低预期，$L$ = 低预期，$M$ = 中预期，$H$ = 高预期，$VH$ = 极高预期。

矩阵揭示了若干可检验的模式化假设。**首先**，方法描述单元格在准确性和规范合规度上呈现最高的能力要求（$VH$），这与其技术文本的精确性本质相一致——任何术语误用或参数错误都可能导致研究不可复现 [R4]。与此同时，该单元格在创造性维度上的预期最低（$VL$），因为方法论写作的核心目标是清晰传递而非创新表达。**其次**，讨论撰写是矩阵中唯一在逻辑连贯性和创造性两个维度上同时要求高水平的子任务——这解释了为什么已有研究指出 LLM 在长文本生成的末端环节容易出现论证松散和观点平庸化的双重问题 [R5]。**再次**，文献综述撰写在准确性和规范合规度上的高预期反映了该子任务对信息忠实度的核心诉求 [R8]。模型在此场景中的主要挑战并非缺乏"创意"，而是能否在大量信息源中准确定位、提取并综合关键论点。

为将矩阵转化为可操作的评估协议，每个单元格需要对应一套具体的测量指标和评分标准。以结果解释单元格（2.3 $\times$ A，即准确性）为例：该单元格的评估操作定义为——将 LLM 生成的结果解释与人工专家标注的"正确解释集"进行比对，统计事实性陈述的错误率 $E = 1 - \frac{n_c}{n_t}$，其中错误类型可进一步细分为数值错误、因果关系误判和过度泛化三类。类似地，讨论撰写单元格（2.4 $\times$ R）的操作定义为——由三位领域专家独立评分 LLM 生成文本的观点新颖性，采用 Likert 五点量表，最终得分取三人评分的均值与标准差，以同时捕捉创造性水平的中心趋势与评价者间分歧程度。

该矩阵框架的理论贡献在于：它将"学术写作能力"从模糊的整体概念拆解为任务特异性和维度特异性的精细组合，从而避免了以往研究中笼统报告 LLM "整体表现良好或不足"的方法论缺陷 [R10]。同时，矩阵的单元级评估协议使得不同研究之间的结果具有可比性——只要采用相同的维度定义和评分标准，任何针对学术写作的 LLM 评测结果都可以映射到该矩阵中进行横向比较。

从分析框架过渡到实证检验，需要回答的核心问题是：上述基于理论推导的预期模式是否能在实际评测中得到验证？换言之，LLM 在十六个单元格中的真实表现水平是否与矩阵预测一致？第三章将围绕这一问题展开系统的实证研究设计，包括评测数据集的构建、评分协议的制定以及初步实验结果的报告。

**本节摘要**：本章构建了"任务类型 $\times$ 能力维度"的 $4 \times 4$ 二维评估矩阵，呈现了每个单元格中 LLM 能力的预期表现水平及其对应的操作化测量方法。该框架为后续实证检验提供了系统化的分析工具和可检验的理论假设。下一章将进入实证研究阶段，报告基于此矩阵的实际评测设计与初步结果。
