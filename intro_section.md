# 1.0 引言

## 1.1 研究背景与问题提出

近年来，以大语言模型（Large Language Model, LLM）为代表的人工智能技术在自然语言处理领域取得了突破性进展。以GPT-4、Claude和文心一言等为代表的系统已具备生成连贯学术文本的能力，其在文献整理、摘要撰写、语言润色等环节的辅助作用日益凸显[3]。多项实证调查显示，超过六成科研工作者曾在某个写作阶段使用过AI工具，其中文献综述和方法描述是最常见的应用场景[10]。然而，实践中普遍存在"信息过载"现象：学术界和工业界推出了数百款AI辅助写作工具，功能定位与能力边界各不相同，一线研究者却难以获得系统化的比较分析以指导工具选择。

现有研究和产品评测多聚焦于单一任务场景——如摘要生成质量、翻译准确度或语言流畅度——缺乏对LLM在学术写作全流程中表现的统一刻画[6]。这种碎片化的认知模式导致两个实际问题：其一，研究者在高认知负荷环节（如讨论撰写、创新性论点提出）过度依赖AI，可能引入引用造假、逻辑幻觉等学术风险；其二，在低认知负荷环节（如文献摘要提取、语法校正）则未能充分利用AI的效能优势，造成科研时间资源的浪费。换言之，当前"AI+学术写作"的实践先于理论认知，亟需一套系统化的能力评估框架以填补这一知识空白。

## 1.2 研究贡献与文章结构

针对上述问题，本文旨在回答以下核心研究问题：大语言模型在学术写作的不同子任务中表现出怎样的效能差异？其能力的适用边界在哪里？为回应这一问题，本文的主要贡献体现在三个方面：第一，提出面向学术写作的"任务类型×能力维度"二维评估矩阵，将学术写作分解为文献综述、方法描述、结果解释和讨论撰写四类子任务，并定义准确性、逻辑连贯性、学术规范合规度和创造性四个能力评估维度；第二，构建标准化评测集与人工双盲评分协议，为后续AI辅助科研工具的效能比较提供可复现的方法论框架；第三，基于实证评估结果识别LLM的高能效与低能效区域，面向一线科研工作者提出分层使用建议与伦理边界规范。

本文结构如下：第二部分建立概念框架，完成学术写作任务分类与LLM能力维度的操作化定义；第三部分报告实证检验的设计与结果；第四部分讨论模型能力的边界条件、适用场景及伦理考量；第五部分总结全文发现并展望研究方向。

**本节摘要**：引言交代了LLM在学术写作中广泛应用的背景与现实痛点——研究者缺乏系统化的AI能力认知与工具选择依据。本文提出"任务类型×能力维度"二维评估框架，构建标准化评测协议，旨在识别LLM在不同写作子任务中的效能差异模式，为一线科研工作者提供基于证据的AI使用指南。
