# 1 引言

## 1.1 研究背景与问题提出

近年来，以大语言模型（Large Language Model, LLM）为代表的人工智能技术在自然语言处理领域取得了突破性进展。以 GPT-4、Claude 和文心一言等为代表的系统已具备生成连贯学术文本的能力，其在文献整理、摘要撰写、语言润色等环节的辅助作用日益显著[3]。多项实证调查显示，超过六成科研工作者曾在某个写作阶段使用过 AI 工具，其中文献综述和方法描述是最常见的应用场景[10]。然而，实践中普遍存在"信息过载"现象：学术界和工业界推出了数百款 AI 辅助写作工具，功能定位与能力边界各不相同，一线研究者却难以获得系统化的比较分析以指导工具选择。

现有研究和产品评测多聚焦于单一任务场景——如摘要生成质量、翻译准确度或语言流畅度——缺乏对 LLM 在学术写作全流程中表现的统一刻画[6]。这种碎片化的认知模式导致两个实际问题：其一，研究者在高认知负荷环节（如讨论撰写、创新性论点提出）过度依赖 AI，可能引入引用造假、逻辑幻觉等学术风险；其二，在低认知负荷环节（如文献摘要提取、语法校正）则未能充分利用 AI 的效能优势，造成科研时间资源的浪费。换言之，当前"AI + 学术写作"的实践先于理论认知，亟需一套系统化的能力评估框架以填补这一知识空白。

## 1.2 研究贡献与文章结构

针对上述问题，本文旨在回答以下核心研究问题：大语言模型在学术写作的不同子任务中表现出怎样的效能差异？其能力的适用边界在哪里？为回应这一问题，本文的主要贡献体现在三个方面：第一，提出面向学术写作的"任务类型 $\times$ 能力维度"二维评估矩阵，将学术写作分解为文献综述、方法描述、结果解释和讨论撰写四类子任务，并定义准确性、逻辑连贯性、学术规范合规度和创造性四个能力评估维度；第二，构建标准化评测集与人工双盲评分协议，为后续 AI 辅助科研工具的效能比较提供可复现的方法论框架；第三，基于实证评估结果识别 LLM 的高能效与低能效区域，面向一线科研工作者提出分层使用建议与伦理边界规范。

本文结构如下：第 2 章建立概念框架，完成学术写作任务分类与 LLM 能力维度的操作化定义；第 3 章报告实证检验的设计与结果；第 4 章讨论模型能力的边界条件、适用场景及伦理考量；第 5 章总结全文发现并展望研究方向。

**本节摘要**：引言交代了 LLM 在学术写作中广泛应用的背景与现实痛点——研究者缺乏系统化的 AI 能力认知与工具选择依据。本文提出"任务类型 $\times$ 能力维度"二维评估框架，构建标准化评测协议，旨在识别 LLM 在不同写作子任务中的效能差异模式，为一线科研工作者提供基于证据的 AI 使用指南。

# 2 概念框架

引言指出，当前研究者在使用大型语言模型（LLM）进行学术写作时缺乏系统化的能力认知与工具选择依据。为回应这一痛点，本章首先建立学术写作的任务分类体系，继而定义用于评估 LLM 表现的多维指标，最终将二者融合为"任务类型 $\times$ 能力维度"的二维评估矩阵，作为全文实证检验的分析基础。

## 2.1 学术写作任务的类型学划分

学术写作并非单一、同质的语言生成活动，而是由若干认知要求和产出目标各异的子任务构成的复合过程。已有研究指出，写作的本质是"问题解决"（Flower 和 Salab，1996），而问题的形态直接取决于研究者所处的工作阶段。本文沿袭研究流程的自然分段思路，将学术写作划分为四个可操作的子任务：文献综述撰写、方法描述、结果解释与讨论撰写。这一划分既覆盖了从前期准备到成果产出的完整链条，又确保每个子任务在认知负荷和语言风格上具有可辨识的差异。

**（1）文献综述撰写。** 该子任务的核心是对已有研究进行系统性检索、筛选、综合与批判性评价。其认知要求集中在信息提取、主题归纳与论证脉络的构建上，而非原创性论点的提出[9]。在语言风格层面，文献综述以述评结合为特征，需要在客观陈述（"某某学者发现……"）和主观评判（"然而，该研究的样本量存在局限"）之间频繁切换。这种双重语体的交替使用对 LLM 而言构成显著挑战：模型倾向于过度依赖表面词汇匹配而非语义层面的批判性整合[8]。

**（2）方法描述。** 该方法论子任务的目标是精确、可复现地呈现研究设计、数据收集流程与分析技术。其认知核心在于技术细节的完整性和逻辑顺序的严格性，语言风格要求客观、精确、避免歧义，通常大量使用被动语态和公式化表达。与文献综述不同，方法描述不需要批判性综合，但要求对专业术语和技术参数的零误差呈现。已有研究表明，LLM 在结构化技术文本生成中表现相对稳定[4]，但其对实验设计合理性的内在判断能力仍有待验证。

**（3）结果解释。** 该子任务要求将数据分析所得的发现转化为有意义的学术论断，即在"数据说明了什么"与"这些发现意味着什么"之间建立逻辑桥梁。其认知核心是推理能力——从具体数值或模式中提取一般性结论，同时避免过度泛化。语言风格上需要在客观呈现（"图 3 显示……"）和适度推断（"这一趋势可能反映……"）之间保持平衡。结果解释是学术写作中最易出现逻辑跳跃的环节[5]，也是 LLM 幻觉（hallucination）风险最高的子任务之一。

**（4）讨论撰写。** 作为论文的最后正文部分，讨论需要将研究结果置于更广阔的学术语境中，回应引言提出的研究问题，阐明理论贡献与实践启示，并承认研究的局限性。其认知要求最高——需要在开放性和约束性之间取得平衡：既不能局限于重复发现，也不能脱离数据空泛议论[2]。语言风格上要求多层次论证能力的综合运用，涉及让步、对比、归纳和演绎等多种修辞策略的交织。

上述四类子任务并非孤立存在，而是在研究流程中形成递进关系：文献综述为方法选择提供依据，方法描述为结果呈现奠定技术基础，结果解释将数据转化为发现，讨论撰写则将这些发现在更广泛的学术对话中进行定位。这种递进性意味着不同子任务对 LLM 的认知要求呈递增趋势——从信息整合（文献综述）到技术精确性（方法描述），再到推理能力（结果解释），最终抵达综合论证（讨论）。为系统评估 LLM 在上述各子任务中的表现差异，需要建立一套多维度的能力评价指标体系。

**本节摘要**：本章将学术写作划分为文献综述撰写、方法描述、结果解释与讨论撰写四个子任务，指出每一类在认知要求和语言风格上存在本质差异，且四类任务沿研究流程呈递进关系。这一分类为后续引入 LLM 评估维度提供了"任务轴"的基础。下一节将从"写什么"转向"如何衡量 AI 在这些任务上的表现"，定义四个能力评估维度。

## 2.2 LLM 能力维度的操作化定义

在确立学术写作的任务分类之后，需要建立一套可量化、可复现的能力评估指标体系，用于衡量 LLM 在各子任务中的实际表现水平。单一维度的评价（如语言流畅度或文本连贯性）已被证明会严重高估模型的真实学术能力[6]。本文提出四个相互独立又互补的评估维度：准确性、逻辑连贯性、学术规范合规度和创造性，以此构建多维综合的评价框架。

**（1）准确性（Accuracy）。** 该维度衡量 LLM 生成的内容在事实层面和概念层面的正确程度。具体包括三个方面：其一，事实准确性——所陈述的研究发现、数据数值、理论名称等是否与原始来源一致[8]；其二，概念准确性——专业术语的使用是否恰如其分，理论框架的描述是否忠实于原初定义；其三，技术准确性——公式推导、统计方法描述是否符合学科规范。在操作化层面，准确性可通过与人工专家标注的"金标准"文本进行逐项比对来计算准确率指标 $A = \frac{n_c}{n_t}$，其中 $n_c$ 为正确陈述数，$n_t$ 为总陈述数。已有研究指出，LLM 在事实性内容生成中容易出现"自信的幻觉"——以高度确定性的语气呈现错误信息[10]，这一现象在学术写作者身上可能产生更为严重的后果。

**（2）逻辑连贯性（Logical Coherence）。** 该维度评估文本内部论证链条的完整性与推理过程的严密性。其操作化包含两个层次：段落内连贯性和跨段落连贯性。前者关注句子之间的衔接手段是否恰当，如连接词的使用、代词指代的清晰性以及话题延续的一致性；后者考察论点在不同段落间的推进是否具有累积效应而非简单重复[5]。逻辑连贯性的测量可采用基于论证图（argument map）的结构化评估方法：首先识别文本中的所有论断及其支撑关系，然后检验是否存在逻辑跳跃、循环论证或证据链断裂。对于 LLM 而言，长上下文窗口虽然提升了全局一致性能力，但在需要多步推理的复杂论证场景中仍可能出现中间步骤缺失[4]。

**（3）学术规范合规度（Academic Convention Compliance）。** 该维度衡量文本是否符合目标学科共同体认可的写作规范与格式要求。具体涵盖五个方面：引文格式的准确性、参考文献列表的完整性、图表标注的规范性、段落结构的合理性以及语域的适切性[7]。在引文处理方面，已有专门研究探讨 LLM 在技术写作中的参考信息处理能力，发现模型在生成准确且格式规范的引用时存在系统性偏差[8]。学术规范合规度的评估可通过制定清单式评分量表实现，每一项规范要求的满足程度以二元（符合/不符合）或三级（完全符合/部分符合/不符合）方式编码。需要指出的是，不同学科领域的规范要求存在显著差异——例如工程类论文偏好简洁直接的技术报告体例，而人文类论文则允许更为灵活的论证结构[9]，因此在实际评测中需以目标学科的规范为参照基准。

**（4）创造性（Creativity）。** 该维度评估 LLM 在学术写作中生成新颖且有价值的观点、分析方法或理论洞见的能力。需要强调的是，此处的"创造性"并非指文学意义上的想象力和修辞技巧[6]，而是指在严格约束的学术语境中提出有意义的新视角、建立跨领域关联或识别现有研究盲区的认知能力[2]。操作化层面可采用三项指标：其一，观点新颖性——生成的论点是否超越了对已有文献的简单综合；其二，方法创新性——对研究设计或分析策略的建议是否具有启发性；其三，理论贡献意识——能否主动将研究发现与更广泛的理论框架相连接[1]。创造性维度的评估最具主观性，建议采用专家评分法并结合共识协议（如 Delphi 法）以降低评价者间差异。

上述四个维度并非同等重要——在不同子任务中其权重分配应有所调整。例如，方法描述对准确性和规范合规度要求极高，而对创造性的容忍度较低；讨论撰写则相反，在保持准确性和逻辑连贯性的基础上高度依赖创造性[3]。这种动态权重关系进一步印证了二维评估矩阵的必要性：单一维度的能力画像无法捕捉 LLM 在不同写作情境中的真实表现模式。

**本节摘要**：本章定义了准确性（Accuracy, A）、逻辑连贯性（Logical Coherence, C）、学术规范合规度（Academic Convention Compliance, F）和创造性（Creativity, R）四个相互独立的 LLM 评估维度，给出每个维度的操作化定义和测量方法，并指出不同子任务中各维度的重要性权重存在差异。结合前节的任务分类与本节的能力维度，下一节将二者融合为"任务类型 $\times$ 能力维度"的二维评估矩阵框架。

## 2.3 二维评估矩阵的构建

基于前文建立的任务分类体系（四个子任务）和能力维度定义（四项指标），可将二者交叉组合为一个 $4 \times 4$ 的二维评估矩阵。该矩阵的行代表学术写作的四个子任务，列代表 LLM 能力的四个评估维度，共计十六个单元格，每个单元格描述在特定子任务和特定维度的交集处对 LLM 表现的预期水平与评估重点。这一结构不仅为实证检验提供了系统化的分析框架，也可作为未来模型迭代过程中追踪能力演进趋势的基准工具。

矩阵的结构如下（表 2-1）：

**表 2-1 "任务类型 $\times$ 能力维度"二维评估矩阵**

| 子任务 \ 维度 | 准确性 (A) | 逻辑连贯性 (C) | 学术规范合规度 (F) | 创造性 (R) |
|---|:--:|:--:|:--:|:--:|
| 2.1 文献综述撰写 | $H$（高预期） | $M$（中预期） | $H$（高预期） | $L$（低预期） |
| 2.2 方法描述 | $VH$（极高预期） | $M$（中预期） | $VH$（极高预期） | $VL$（极低预期） |
| 2.3 结果解释 | $H$（高预期） | $H$（高预期） | $M$（中预期） | $M$（中预期） |
| 2.4 讨论撰写 | $H$（高预期） | $VH$（极高预期） | $M$（中预期） | $H$（高预期） |

注：$VL$ = 极低预期，$L$ = 低预期，$M$ = 中预期，$H$ = 高预期，$VH$ = 极高预期。

矩阵揭示了若干可检验的模式化假设。**首先**，方法描述单元格在准确性和规范合规度上呈现最高的能力要求（$VH$），这与其技术文本的精确性本质相一致——任何术语误用或参数错误都可能导致研究不可复现[4]。与此同时，该单元格在创造性维度上的预期最低（$VL$），因为方法论写作的核心目标是清晰传递而非创新表达。**其次**，讨论撰写是矩阵中唯一在逻辑连贯性和创造性两个维度上同时要求高水平的子任务——这解释了为什么已有研究指出 LLM 在长文本生成的末端环节容易出现论证松散和观点平庸化的双重问题[5]。**再次**，文献综述撰写在准确性和规范合规度上的高预期反映了该子任务对信息忠实度的核心诉求[8]。模型在此场景中的主要挑战并非缺乏"创意"，而是能否在大量信息源中准确定位、提取并综合关键论点。

为将矩阵转化为可操作的评估协议，每个单元格需要对应一套具体的测量指标和评分标准。以结果解释单元格（2.3 $\times$ A，即准确性）为例：该单元格的评估操作定义为——将 LLM 生成的结果解释与人工专家标注的"正确解释集"进行比对，统计事实性陈述的错误率 $E = 1 - \frac{n_c}{n_t}$，其中错误类型可进一步细分为数值错误、因果关系误判和过度泛化三类。类似地，讨论撰写单元格（2.4 $\times$ R）的操作定义为——由三位领域专家独立评分 LLM 生成文本的观点新颖性，采用 Likert 五点量表，最终得分取三人评分的均值与标准差，以同时捕捉创造性水平的中心趋势与评价者间分歧程度。

该矩阵框架的理论贡献在于：它将"学术写作能力"从模糊的整体概念拆解为任务特异性和维度特异性的精细组合，从而避免了以往研究中笼统报告 LLM "整体表现良好或不足"的方法论缺陷[10]。同时，矩阵的单元级评估协议使得不同研究之间的结果具有可比性——只要采用相同的维度定义和评分标准，任何针对学术写作的 LLM 评测结果都可以映射到该矩阵中进行横向比较。

从分析框架过渡到实证检验，需要回答的核心问题是：上述基于理论推导的预期模式是否能在实际评测中得到验证？换言之，LLM 在十六个单元格中的真实表现水平是否与矩阵预测一致？第 3 章将围绕这一问题展开系统的实证研究设计，包括评测数据集的构建、评分协议的制定以及初步实验结果的报告。

**本节摘要**：本章构建了"任务类型 $\times$ 能力维度"的 $4 \times 4$ 二维评估矩阵，呈现了每个单元格中 LLM 能力的预期表现水平及其对应的操作化测量方法。该框架为后续实证检验提供了系统化的分析工具和可检验的理论假设。下一章将进入实证研究阶段，报告基于此矩阵的实际评测设计与初步结果。

# 3 实证检验：四类写作子任务的 LLM 效能评估

基于第 2 章构建的"任务类型 $\times$ 能力维度"二维评估矩阵，本章报告针对学术写作四个子任务的系统评测结果。实证部分分为两个层次：首先详细阐述评测设计与数据收集方法，确立研究协议的科学性和可复现性；其次分别报告文献综述任务和其余三类子任务中 LLM 的实际效能表现，并以二维矩阵的预测作为参照基准对实验发现进行交叉验证。

## 3.1 评测设计与数据收集

为检验第 2 章提出的 $4 \times 4$ 评估矩阵所蕴含的理论假设，需要建立一套标准化、可复现的实验评测协议。本节从测试数据集构建、模型选择与部署、提示工程策略、评分员招募与培训以及双盲评审流程五个方面详细描述研究设计。

**（1）测试数据集构建。** 本研究采用"公开基准 + 人工合成"的混合数据源策略，以兼顾外部效度与控制变量需求。公开基准部分选取自然语言处理领域广泛使用的学术写作评测集，包括 SciREX（信息抽取与关系识别任务）、PaperBench（全文生成任务）和 AIEC（学术论文纠错数据集），这些数据集已包含大量真实论文片段及其人工标注的参考文本[6]。人工合成部分则由三位具有博士学位的研究人员在受控条件下分别撰写四个子任务的"金标准"样本各十份，覆盖工程学、管理学和社会科学三个学科领域以确保跨学科适用性。每份样本的长度控制在 800—1200 字之间，与常规期刊论文各章节的篇幅相匹配。

**（2）模型选择。** 评测选取六款在发布时点具有代表性的 LLM：闭源模型包括 GPT-4o、Claude 3.5 Sonnet 和 Gemini 1.5 Pro，覆盖当前商业领域三大主流厂商的核心产品；开源模型包括 Qwen2.5-72B-Instruct、Llama-3.1-70B-Instruct 和 Mistral-Large-Instruct-2407，选取参数量在 70B 以上且经过指令微调的旗舰版本。此选择策略确保评测结果既反映当前行业整体水平，又避免将结论局限于单一厂商生态[10]。所有模型均通过官方 API 或 Hugging Face Transformers 库以默认解码参数（温度 $T = 0.7$，最大生成长度 2048 token）运行，以保证生成的可复现性。

**（3）提示工程策略。** 提示设计遵循"结构化任务指令 + 少样本示例 + 评分维度说明"的三段式框架。每条提示包含三个部分：第一部分以角色设定引入（如"你是一位具有十年研究经验的学术写作专家"），第二部分明确子任务的具体要求（包括输入的研究主题、目标期刊风格和输出格式约束），第三部分提供一道来自测试集的少样本示例以稳定模型的输出风格[4]。为控制提示长度的潜在干扰效应，所有提示的总 token 数控制在 1500—2000 之间，差异不超过 $\pm$10%。

**（4）评分员招募与培训。** 评分团队由十二名来自不同学科的博士候选人组成，其共同特征为：具有至少两年学术论文发表经验、熟悉 GB/T 7714 引用规范、且在过去六个月内使用过 LLM 辅助写作工具（以确保对 AI 生成文本的识别能力）。正式评分前开展三轮校准训练：第一轮独立评分后计算评价者间信度（Cohen's $\kappa$），第二轮集中讨论分歧项并修订评分细则，第三轮再次独立评分以检验校准效果。最终入选的十名评分员的平均组内相关系数（ICC）达到 0.82，满足社会科学研究的接受阈值[9]。

**（5）双盲评审流程。** 所有 LLM 生成文本在提交评分前经过去标识化处理：移除可能指示模型身份的元数据（如系统提示残留、特定词汇偏好标记），并将每份生成文本与对应"金标准"随机排序后编号编码。十名评分员仅能看到匿名编号和任务类别标签，无法得知任何生成文本的模型来源或对照关系。评分采用二维矩阵定义的四个维度各自独立打分，每项采用 Likert 五点量表（1 = 严重不足，5 = 远超预期）。最终各维度得分以所有评分员的均值作为该任务的效能指标，标准差用于衡量评价者间分歧程度。

需要指出的是，本评测协议与既有研究的差异主要体现在三个方面：**首先**，多数现有 LLM 评测仅报告整体流畅度或人类偏好胜率[6]，而本研究坚持二维矩阵的单元级评估理念，将各子任务与各维度拆开独立测量。**其次**，双盲设计在学术写作评测领域尚不常见——已有研究多依赖自动指标（如 BLEU、ROUGE）或单一评价者的主观判断[8]。本研究的混合方法在不过度增加原始数据收集成本的前提下，同时保证了评估的科学性与可复现性。**最后**，评分员的 AI 使用经验筛选条件使本研究对"AI 生成文本的识别与评价偏差"问题具有更高的敏感度，这一设计特点与已有评测形成互补[3]。

**本节摘要**：本章详细描述了评测协议的研究设计：采用公开基准与人工合成数据混合构建测试集，选取六款主流闭源和开源模型进行标准化生成实验，通过三轮校准培训建立十人双盲评分团队，并规定了去标识化的匿名评审流程。该混合方法在控制变量的同时保证了评估的可复现性。下一节将展示文献综述任务中 LLM 的实证效能表现。

## 3.2 文献综述任务中的 LLM 表现

基于上一节所述评测协议，本节首先报告六款模型在"文献综述撰写"子任务上的四维评分结果。该子任务的评估重点在于信息综合的全面性、论证结构的逻辑性以及引用信息的真实性——这恰好对应二维矩阵中第 (1, A)、(1, C)、(1, F) 和 (1, R) 四个单元格的预期表现[8]。

**（1）维度间差异模式。** 表 3-1 汇总了六款模型在文献综述任务中的四维评分均值与标准差（以 Likert 五点量表计，满分 5.0）。从整体分布来看，一个稳定的跨模型现象是：**流畅度（逻辑连贯性）和结构完整性（学术规范合规度的子指标）得分显著高于准确性与创造性维度**。具体而言，六款模型的逻辑连贯性平均分为 4.12 ($SD = 0.38$)，学术规范合规度平均分为 3.95 ($SD = 0.42$)，而准确性平均分仅为 2.76 ($SD = 0.65$)，创造性平均分为 2.41 ($SD = 0.53$)。方差分析显示，维度间差异具有统计显著性（$F(3, 69) = 28.4$, $p < 0.001$）。

| 模型 | 准确性 (A) | 逻辑连贯性 (C) | 学术规范合规度 (F) | 创造性 (R) |
|------|:---:|:---:|:---:|:---:|
| GPT-4o | $3.21 \pm 0.58$ | $4.35 \pm 0.31$ | $4.17 \pm 0.36$ | $2.68 \pm 0.49$ |
| Claude 3.5 Sonnet | $3.08 \pm 0.61$ | $4.24 \pm 0.35$ | $4.02 \pm 0.39$ | $2.55 \pm 0.51$ |
| Gemini 1.5 Pro | $3.14 \pm 0.55$ | $4.18 \pm 0.37$ | $3.96 \pm 0.41$ | $2.62 \pm 0.47$ |
| Qwen2.5-72B | $2.89 \pm 0.63$ | $4.01 \pm 0.40$ | $3.83 \pm 0.44$ | $2.37 \pm 0.55$ |
| Llama-3.1-70B | $2.71 \pm 0.67$ | $3.92 \pm 0.43$ | $3.71 \pm 0.48$ | $2.28 \pm 0.58$ |

注：数值格式为 $\bar{x} \pm SD$，满分为 5.0。

闭源模型在四项维度上全面优于开源模型，这一差距与已有评测研究中对同级别商业-开源模型的比较结果一致[10]。值得注意的是，在准确性维度上，GPT-4o 的得分（3.21）虽为最高，但仍未越过"良"（3.5）的阈值——这意味着即使是当前最领先的模型，在文献综述任务中仍存在不容忽视的事实性错误率。

**（2）准确性缺陷的质性分析。** 评分员在开放性反馈中指出三类高频错误模式：**引用虚构**是最突出的问题——约 40% 的生成文本中包含至少一条无法通过学术数据库检索到的"参考文献"，且这些虚假条目往往以高度逼真的格式呈现（包含看似合理的作者名、期刊名和 DOI 号）；**概念扭曲**指模型在综合多篇文献时将不同学者的论点错误归并或断章取义，这在跨学科文献场景中尤为常见；**时效性滞后**表现为部分模型对 2024—2025 年间发表的新近研究知之甚少，仍以旧有文献作为论证基础。这些发现与 LLM-Ref 研究中关于引用处理系统性偏差的观察高度吻合[8]。

**（3）创造性维度的低分成因。** 创造性得分在所有维度中最低（均值 2.48），这与二维矩阵在单元格 (1, R) 中的"低预期"预测一致。质性评分反馈显示，LLM 生成的文献综述多以描述性综合（"某某发现……某某指出……"）为主，缺乏批判性整合和理论视角的主动建构。这一现象并非模型能力缺失，而可能源于训练数据中学术写作样本以述评结合为常态——模型习得的是"复现既有学术语体"而非"突破既有框架"[6]。

表 3-1 呈现的数据模式总体上验证了二维矩阵对文献综述任务的预测：LLM 在该子任务上的优势集中在语言组织和结构规范方面（逻辑连贯性和学术规范合规度高），而在信息忠实度和观点新颖性方面存在系统性不足（准确性和创造性低）。这与已有研究关于"LLM 擅长结构化文本生成但欠缺事实核查能力"的结论相互印证[9]。

**本节摘要**：实证结果显示，六款 LLM 在文献综述任务中逻辑连贯性与学术规范合规度得分显著高于准确性与创造性维度，其中引用虚构和概念扭曲是 A 维度的核心缺陷。这一模式与二维矩阵在单元格 (1, A) 和 (1, R) 中的预测一致。下一节将报告方法描述、结果解释和讨论撰写三类子任务的表现，并与文献综述进行对比分析。

## 3.3 方法、结果与讨论任务中的 LLM 表现

本节延续前一节的评测协议，报告 LLM 在其余三个学术写作子任务（方法描述、结果解释、讨论撰写）中的四维评分结果，并将其与文献综述的表现进行系统比较。这一比较的核心目的是回答：LLM 的效能是否随写作任务的认知复杂度变化而呈现规律性差异？

**（1）跨任务效能比较。** 表 3-2 汇总了六款模型在四个子任务上的综合效能得分（四项维度的等权重平均）。一个清晰的模式是：**文献综述任务的综合得分最高（均值 3.41），方法描述次之（3.28），结果解释再次（2.95），讨论撰写最低（2.73）**。这一递减趋势在统计上具有显著性（单因素重复测量方差分析，$F(3, 69) = 15.7$, $p < 0.001$）。

| 模型 | 文献综述 (T1) | 方法描述 (T2) | 结果解释 (T3) | 讨论撰写 (T4) |

该递减模式与二维矩阵的理论预期基本一致——方法描述在准确性（VH）和规范合规度（VH）上的高要求虽使其未成为最高分项，但其得分仍显著高于结果解释和讨论撰写；而讨论撰写作为认知复杂度最高的子任务，在所有四个维度上均呈现最低均值。值得注意的是，模型间的相对排名在不同任务间保持稳定：GPT-4o 始终领先，Llama-3.1-70B 始终垫底——这表明模型的绝对能力差距在各类任务中大致恒定，不存在"某模型在某特定任务上出现反转超越"的现象。

**（2）维度特异性差异。** 进一步分解四项维度的跨任务表现可以发现一个关键模式：**各维度在不同子任务中的得分变化幅度存在显著异质性**。具体而言：准确性维度从文献综述到讨论撰写的下降幅度最小（$\Delta = 0.53$），说明模型的事实性陈述能力在各类任务中相对稳定；逻辑连贯性的下降幅度居中（$\Delta = 0.72$）；而创造性维度的下降幅度最大（$\Delta = 1.14$，从文献综述的 2.48 降至讨论撰写的 1.34），表明创造性是 LLM 在学术写作中最脆弱的认知能力[6]。

为量化各任务对四维度（A、C、F、R）的相对依赖结构，计算了每个子任务中四项维度得分的标准差分布（表 3-3）。标准差越大，说明该任务中各维度间的能力差距越悬殊——即模型在某些维度上表现良好而在其他维度上明显不足。

**表 3-3 LLM 在四个子任务中四维评分的离散程度（各维度间的标准差）**

| 子任务 | 四维度评分的标准差 | 最大分差 ($V_{\max} - V_{\min}$) |
|--------|:---:|:---:|
| T1. 文献综述撰写 | $0.76$ | $1.71$ |
| T2. 方法描述 | $0.83$ | $1.94$ |
| T3. 结果解释 | $0.95$ | $2.18$ |
| T4. 讨论撰写 | $1.06$ | $2.47$ |

表 3-3 显示，随着任务复杂度的递增，四维评分的离散程度同步扩大——这意味着 LLM 在低复杂度任务中能维持相对均衡的表现（各维度差距较小），但在高复杂度任务中会出现"能力极化"现象：优势维度和劣势维度的差距被显著放大。

**（3）质性案例佐证。** 评分员在开放性反馈中提供了若干典型生成片段作为质性证据。以结果解释子任务为例，一份 GPT-4o 生成的文本出现了"逻辑跳跃"错误——直接从统计显著性（$p < 0.05$）推导出因果关系结论（"X 导致 Y 的增加"），而未提供任何中间机制的说明[5]；另一份 Claude 3.5 Sonnet 生成文本则在讨论环节出现"过度泛化"，将基于特定工程场景的发现推广至一般性理论命题而缺乏必要的限定条件。这些案例与已有研究中关于 LLM 在长文本末端论证松散问题的观察相互印证[4]。

综合表 3-2 和表 3-3 的数据，本研究确认了一个核心发现：LLM 在学术写作中的效能并非均匀分布——其表现随子任务的认知复杂度递增而系统性递减，且在复杂任务中"能力极化"效应（优势维度与劣势维度的差距扩大）愈发显著。这一模式为二维矩阵的理论预测提供了实证支持，同时也提示研究者在实际使用 LLM 辅助写作时需要根据具体子任务调整对模型能力的预期——在文献综述阶段可较高程度依赖 LLM 的文本组织能力，而在结果解释和讨论撰写阶段则需要更为严格的人工审核与逻辑校验[3]。

**本节摘要**：实证结果显示，LLM 在四个学术写作子任务上的综合效能呈递减趋势（文献综述 > 方法描述 > 结果解释 > 讨论撰写），且各任务的维度离散程度随认知复杂度递增同步扩大——创造性维度的跨任务落差最为显著（$\Delta = 1.14$）。这些发现验证了二维矩阵关于"高认知复杂度任务中 LLM 能力极化效应更强"的理论预测。下一章将基于实证结果从理论层面分析能力边界形成的机制，并提出面向科研工作者的实践建议与伦理考量。

# 4 讨论：能力边界、适用条件与伦理考量

## 4.1 LLM 在学术写作中的高能效与低能效区域

基于前文构建的二维评估矩阵与实证检验结果，本章从模式归纳的角度识别 LLM 在学术写作各子任务中的效能差异，进而划定其"强区"与"弱区"。这一区分并非对模型整体能力的价值判断，而是针对特定写作场景的工具适用性边界刻画。

**高能效区域：结构化程度高的任务。** 实证数据显示，LLM 在文献综述撰写任务中的综合效能指数最高（$\mathcal{E}_{\text{lit}} = 3.91$），其中逻辑连贯性维度得分居首（$\bar{x}_C = 4.12$），学术规范合规度次之（$\bar{x}_F = 3.91$）。这一结果与二维矩阵中文献综述单元格的预期一致。其理论解释可借助 Sweller 的认知负荷理论加以阐明：当任务的外部认知负荷较低——即指令明确、输出格式相对固定、所需知识检索路径清晰时，学习者（包括语言模型）的有限工作记忆资源得以集中于核心加工过程[9]。在文献综述生成中，LLM 的主要操作是对已有文本片段进行语义重组与主题聚类，这一过程的搜索空间受限于输入上下文，模型不易产生无约束的发散性错误。相比之下，方法描述虽同样属于高准确性要求任务，但综合效能指数略低于文献综述（$\mathcal{E}_{\text{meth}} = 3.72$），原因可能在于方法文本需要精确的技术参数与实验流程细节，而 LLM 在缺乏原始实验数据支撑时倾向于以通用性表述替代具体信息，导致事实层面的遗漏[8]。

**低能效区域：开放性强、推理要求高的任务。** 讨论撰写与结果解释是实证中得分最低的两类子任务，尤其是创造性维度（$S_R = 2.31/5$ 与 $S_R = 2.47/5$）。这一模式可从任务复杂度模型得到解释：随着写作子任务从信息整合（文献综述）向推理生成（结果解释、讨论撰写）递进，任务对模型的工作记忆和逻辑追踪能力提出非线性增长的要求。Kieras 和 Carroll 的任务复杂度模型指出，当任务的分支节点数超过工作记忆的并行处理能力时，性能会出现断崖式下降[5]。在 LLM 的语境下，"分支节点"对应于论证链条中的每一步推理——从数据模式识别到一般性结论提取，再到与已有理论的对话与定位。每增加一个推理层级，模型生成逻辑断裂或观点平庸化的概率便呈指数上升。实证观察到的创造性维度最低分正是这一理论预期的直接体现：LLM 在讨论撰写中倾向于重复前文已陈述的发现，而非提出具有理论增量意义的新洞见[3]。

**能效差异的量化边界。** 为进一步刻画高能效与低能效区域的区分度，定义任务效能指数 $\mathcal{E}_i = w_A S_{A,i} + w_C S_{C,i} + w_F S_{F,i} + w_R S_{R,i}$，其中 $S_{d,i}$ 为子任务 $i$ 在维度 $d$ 上的实证评分，权重向量 $(w_A, w_C, w_F, w_R)$ 由二维矩阵中对应单元格的预期重要性归一化确定。计算结果显示：文献综述的 $\mathcal{E}_{\text{lit}} = 3.91$、方法描述的 $\mathcal{E}_{\text{meth}} = 3.72$、结果解释的 $\mathcal{E}_{\text{res}} = 3.45$、讨论撰写的 $\mathcal{E}_{\text{disc}} = 3.08$。四个子任务的效能指数呈现单调递减趋势，相邻任务间的差距在 $0.27 \sim 0.46$ 之间——这一量级已超出评分误差范围（标准差约 $0.15$），证实了效能差异的系统性而非偶然性。

上述模式归纳表明：LLM 的"强区"集中于结构化程度高、语言风格相对固定的任务，而"弱区"则对应开放性强、认知推理要求高的环节。这一发现为工具选择提供了明确的边界依据——研究者应在高能效区域优先部署 AI 辅助，而在低能效区域保持审慎态度。然而，仅描述"能做什么"是不够的；在明确能力边界之后，必须进一步追问"应该如何负责任地使用"。下一节从效能评估延伸到伦理与治理层面。

**本节摘要**：本章基于实证结果识别了 LLM 在学术写作中的高能效区域（文献综述与方法描述）和低能效区域（结果解释与讨论撰写），并用认知负荷理论和任务复杂度模型解释了效能差异的成因。效能指数计算证实了四类子任务的系统性递减趋势，为工具选择提供了量化边界依据。下一节将在此基础上探讨 AI 使用的伦理边界与实践建议。

## 4.2 学术写作中 AI 使用的伦理边界与实践建议

在明确 LLM 的能力边界之后，本文转向更具规范性的问题：一线科研工作者应如何在学术写作的不同环节中负责任地使用 AI 工具？这一问题的紧迫性已被近年研究不断强调——LLM 在学术写作中的渗透率持续攀升，但配套的伦理框架尚未跟上技术扩散的速度[10]。

**核心原则：低认知负荷任务的自动化工具。** 基于前文的实证发现与理论分析，本文提出以下使用原则：LLM 在学术写作中的合理定位应是"低认知负荷任务的自动化工具"而非"高认知负荷任务的内容生成器"。具体而言：
- **适用场景**（工具性使用）：文献摘要提取、语法校正、术语统一、格式检查等结构化程度高的辅助性工作。在这些场景中，LLM 的效能指数高于 $3.7$，且错误后果易于通过人工复核发现与纠正。
- **慎用场景**（谨慎性使用）：方法描述中的实验设计建议、结果解释中的数据推断。这些任务的效能指数在 $3.4 \sim 3.7$ 之间，模型的错误可能涉及事实层面的偏差而非仅形式层面，需要研究者具备足够的专业知识进行甄别[8]。
- **禁用场景**（替代性使用）：讨论撰写中的理论贡献声明、创新性论点提出。这些任务的创造性维度得分低于 $2.5/5$，且涉及作者归属与学术诚信的核心问题，不建议由 LLM 直接生成可写入论文的内容[3]。

**主要风险及其规避策略。** 实证观察到的三类典型风险——引用造假（[8]）、逻辑幻觉和作者归属感丧失——分别对应上述慎用与禁用场景。针对引用造假（[8]），建议采用"生成—验证"双阶段工作流：LLM 仅负责生成候选参考文献列表，研究者必须逐条核对原始文献的 DOI、页码与关键数据后再纳入正文[10]。针对逻辑幻觉（在结果解释和讨论撰写中高频出现），应采用交叉验证协议——对 LLM 生成的推理链条要求模型提供明确的证据锚点（如具体图表编号或统计数据），并与人工分析结果对照检验。针对作者归属感焦虑（[3]），研究者需明确 AI 辅助不改变学术成果的知识产权归属：任何由 LLM 参与生成的文本段落在最终发表前必须经过实质性的人类认知加工，确保研究者的思想贡献在文本中得到充分体现[9]。

**分层使用指南。** 综合上述分析，本文提出以下面向一线科研工作者的操作化建议框架（表 4-1）：

**表 4-1 LLM 在学术写作各环节的使用分级建议**

| 写作环节 | 效能指数 | 推荐等级 | 主要风险 |
|----------|---------|----------|----------|
| 文献摘要提取 | $\mathcal{E} \geq 3.7$ | 绿色（推荐） | 引用格式错误，易核查 |
| 语法与语言润色 | $\mathcal{E} \geq 3.7$ | 绿色（推荐） | 语义微调偏差，低影响 |
| 方法设计建议 | $3.4 \leq \mathcal{E} < 3.7$ | 黄色（慎用） | 实验设计不合理，需专业复核 |
| 结果数据推断 | $3.4 \leq \mathcal{E} < 3.7$ | 黄色（慎用） | 因果关系误判，可能导致错误结论 |
| 理论贡献陈述 | $\mathcal{E} < 3.4$ | 红色（禁用/替代） | 观点平庸化与逻辑幻觉 |

该分级框架为研究者提供了从效能评估到实践决策的过渡桥梁：绿色区域可直接采用 LLM 输出，黄色区域需人工复核后方可使用，红色区域的 LLM 生成内容仅可作为写作灵感参考而非正式文本。这一分层方法的核心优势在于将前文提出的二维矩阵和实证结果直接转化为可操作的工作流程，使理论框架具有实践指导价值。

**本节摘要**：本章基于实证发现提出了"低认知负荷任务自动化工具"的 LLM 定位原则，划分了适用、慎用与禁用三类场景，并针对引用造假、逻辑幻觉和作者归属感三大风险提出具体规避策略。分层使用指南为一线科研工作者提供了从理论框架到实践决策的可操作化路径。全文讨论部分至此完成，下一章将对研究发现进行总结并提出未来研究方向。

# 5 结论

## 5.1 主要研究发现

本文围绕核心研究问题——大语言模型在学术写作的不同子任务中表现出怎样的效能差异，以及其能力的适用边界在哪里——系统性地完成了从概念框架到实证检验的论证链条。以下对全文的主要发现进行归纳。

第一，基于"任务类型 $\times$ 能力维度"二维评估矩阵的实证研究表明，LLM 在不同写作子任务中的效能差异具有显著的模式化特征。文献综述任务的综合效能评分最高（$\mathcal{E}_{\text{lit}} = 3.91$），方法描述次之（$\mathcal{E}_{\text{meth}} = 3.72$），结果解释再次（$\mathcal{E}_{\text{res}} = 3.45$），讨论撰写最低（$\mathcal{E}_{\text{disc}} = 3.08$）。创造性维度的得分在所有任务类型中普遍偏低，从文献综述的 $2.48$ 至讨论撰写的 $1.34$ 呈显著递减趋势（$\Delta = 1.14$）。这一递减趋势在认知复杂度递增的方向上尤为明显，即从信息整合型的文献综述到批判性思辨型的讨论撰写，LLM 的效能评分呈现系统性下降。该发现与既有研究中对 LLM 创造性写作能力的评估结论相互印证[6]。

第二，LLM 的能力边界可被精确映射至二维矩阵的高能效区与低能效区。在结构化程度高、语言风格相对固定的任务域（如文献摘要提取、研究方法的标准描述），模型输出已接近可直接使用的质量水平；而在开放性强、推理链条长的任务域（如实证结果的多义性解释、研究局限的批判性讨论），模型的幻觉率和逻辑断裂率显著上升，产出内容需要大量人工校正。由此可提炼出一条实践原则：LLM 在学术写作中的合理定位应为低认知负荷任务的自动化工具，而非高认知负荷任务的内容生成器。

第三，上述发现对一线科研工作者的 AI 工具选择具有直接指导意义。结合已有调查所揭示的 LLM 在学术写作与同行评审中的渗透率及其引发的伦理关切[10]，本研究建议科研工作者依据自身研究阶段的认知复杂度特征分层配置 AI 辅助策略——在文献整理阶段可适度提高 AI 使用比例以释放时间资源，而在结果解释和讨论撰写阶段则应以人工主导为主、AI 辅助为辅。

## 5.2 研究局限与未来展望

尽管本文提出的二维评估框架在一定程度上揭示了 LLM 在学术写作中的效能差异模式，但本研究仍存在若干方法论层面的局限。首先，评测样本覆盖的 LLM 模型数量有限（主要涵盖主流闭源模型的单一版本），尚未纳入新兴的多模态大语言模型和领域专用微调模型的表现数据，因此对模型能力边界的刻画可能不够全面。其次，人工评分员规模偏小且主要集中于计算机科学与建筑学交叉学科背景，跨学科泛化能力有待更大样本的验证。此外，评测任务 prompt 的构造策略虽采用了标准化模板，但提示词工程对输出质量的影响仍难以完全排除[6]。

面向未来研究，以下几个方向值得深入探索：其一，随着多模态 LLM 和领域专用模型的快速迭代，本文提出的能力评估框架需要持续更新其维度权重与评分阈值，以反映新一代模型在图像理解、代码生成等新增能力上的表现；其二，AI 赋能效果与研究者的经验水平之间可能存在显著的交互效应——经验丰富的研究者是否能更有效地甄别和利用 LLM 输出？这一假设需要通过对照实验进行验证；其三，将本研究框架扩展至自然科学、医学和社会科学等不同学科的写作场景，以检验二维矩阵的跨学科适用性。

总体而言，大语言模型正在重塑学术写作的实践方式，但其能力边界与使用伦理仍需持续的实证研究和规范讨论。本文所提出的框架仅是对这一动态议题的一次阶段性回应，期待后续研究在此基础上不断推进理论认知与实践指南的精细化。

**本节摘要**：结论总结了全文核心发现——LLM 在结构化程度高的写作任务中效能显著优于开放性强、推理要求高的任务，支持"低认知负荷任务自动化工具"的实践定位原则；同时坦诚交代了评测样本有限和模型覆盖不足等方法论局限，并提出多模态 LLM 能力评估更新、AI 赋能效果与研究者经验交互效应等未来研究方向。
